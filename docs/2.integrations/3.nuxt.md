Lighthouse auditing during Nuxt development with automatic route discovery and HMR support.

## Install

::code-group

```bash [yarn]
yarn add -D kumphouse-nuxt
```

```bash [npm]
npm install -D kumphouse-nuxt
```

```bash [pnpm]
pnpm add -D kumphouse-nuxt
```

::

### Git ignore reports

Kumphouse will save your reports in `outputDir` (`.kumphouse` by default),
it's recommended you .gitignore these files.

```shell
.kumphouse
```

## Usage

To begin using Kumphouse, you'll need to add the module to `buildModules`.

When you run your Nuxt app, it will give you the URL of client, only once you visit the client will Kumphouse start.

### Nuxt 3

```js nuxt.config.ts
import { defineNuxtConfig } from 'nuxt3'

export default defineNuxtConfig({
  modules: [
    'kumphouse-nuxt',
  ],
})
```

### Nuxt 2

```js nuxt.config.js
export default {
  buildModules: [
    'kumphouse-nuxt',
  ],
}
```

Type support can be added by adding the `kumphouse-nuxt` module to your `plugins`. Nuxt v3 will automatically add type support.

```json tsconfig.json
{
  "compilerOptions": {
    "types": [
      "kumphouse-nuxt"
    ]
  }
}
```

## Configuration

You can either configure Kumphouse via the `kumphouse` key in your Nuxt config, or you can provide a `kumphouse.config.ts` file
in the root directory.

### Example

```js nuxt.config.js
export default {
  kumphouse: {
    scanner: {
      // simulate a desktop device
      device: 'desktop',
    }
  }
}
```
