## Install

::code-group

```bash [yarn]
yarn add -D kumphouse-vite
```

```bash [npm]
npm install -D kumphouse-vite
```

```bash [pnpm]
pnpm add -D kumphouse-vite
```

::

### Git ignore reports

Kumphouse will save your reports in `outputDir` (`.kumphouse` by default),
it's recommended you .gitignore these files.

```shell
.kumphouse
```

## Usage

To begin using Kumphouse, you'll need to add the plugin to `plugins`.

When you run your Vite app, it will give you the URL of client, only once you visit the client will Kumphouse start.

```ts vite.config.ts
import Kumphouse from 'kumphouse-vite'
import { defineConfig } from 'vite'

export default defineConfig({
  plugins: [
    Kumphouse({
      // config
    })
  ]
})
```

### Providing Route Definitions

If you're using the [vite-plugin-pages](https://github.com/hannoeru/vite-plugin-pages) plugin, you can provide route definitions to Kumphouse.

You will need to hook into the plugin using the following code.

```ts vite.config.ts
import { useKumphouse } from 'kumphouse'
import { defineConfig } from 'vite'

export default defineConfig({
  plugins: [
    Pages({
      // ...
      onRoutesGenerated(routes) {
        // tell Kumphouse about the routes
        const kumphouse = useKumphouse()
        if (kumphouse?.hooks)
          kumphouse.hooks.callHook('route-definitions-provided', routes)
      }
    }),
  ]
})
```

## Configuration

You can either configure Kumphouse via the plugin, or you can provide a  [config file](/guide/guides/config) file
in the root directory.

### Example

```ts vite.config.ts
import Kumphouse from 'kumphouse-vite'
import { defineConfig } from 'vite'

export default defineConfig({
  plugins: [
    Kumphouse({
      scanner: {
        // simulate a desktop device
        device: 'desktop',
      }
    })
  ]
})
```
